# Artifacts

Artifacts are files or data generated by agents during a session (e.g., a generated CSV, a generated image, or a text report). The ADK provides services to save, load, and manage these artifacts.

## Built-in Services

- **`InMemoryArtifactService`**: Stores artifacts in memory. Good for testing.
- **`FileArtifactService`**: Stores artifacts on the local filesystem.
- **`GcsArtifactService`**: Stores artifacts in Google Cloud Storage, supporting versioning.
- **`ForwardingArtifactService`**: Used internally by sub-agents to forward artifact operations to their parent agent's context.

## Example: Save and Load Artifacts

Here is how you can manually save and load an artifact using the `FileArtifactService`.

```csharp
using GoogleAdk.Core.Artifacts;
using GoogleAdk.Core.Abstractions.Models;

// Initialize the service with a base directory
var service = new FileArtifactService("my_artifacts_folder");

// Save an artifact
await service.SaveArtifactAsync(new SaveArtifactRequest
{
    AppName = "my_app",
    UserId = "user_123",
    SessionId = "session_456",
    Filename = "summary_report.txt",
    Artifact = new Part { Text = "This is the final summary." }
});

// Load the artifact later
var loadedPart = await service.LoadArtifactAsync(new LoadArtifactRequest
{
    AppName = "my_app",
    UserId = "user_123",
    SessionId = "session_456",
    Filename = "summary_report.txt"
});

Console.WriteLine(loadedPart?.Text); // Outputs: This is the final summary.
```

## Supplying the Artifact Service to the Runner

You typically configure the artifact service when creating your `Runner`. The runner makes it available to the agent via the `AgentContext`.

```csharp
var runner = new Runner(new RunnerConfig
{
    AppName = "my_app",
    Agent = myAgent,
    ArtifactService = new FileArtifactService("artifacts_dir")
});
```
